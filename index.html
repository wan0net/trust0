trust0

Prove that person X owns account X. Cryptographic identity verification that fills the gap Keybase left — with an append-only sigchain, key rotation, and document signing. Open source.

Documentation GitHub

How it works

  1. Generate an Ed25519 keypair in your browser
  2. Post your fingerprint on GitHub, DNS, Mastodon, or wherever
  3. Add the claim to your signed profile
  4. Anyone can verify — their browser fetches the proof and checks the signature
  5. The server just stores bytes. It can't forge results.

What makes this different

Sigchain

Append-only signed history. See when proofs were added, keys rotated, claims revoked.

Key rotation

Change your key without losing your identity. The sigchain preserves continuity.

Document signing

Sign files with your identity key. Rekor timestamps prove when you signed.

Portable data

Export your identity. Self-host it. Every file is a self-verifying JWS.

Client-side verification

The viewer's browser does all the crypto. No server trust needed.
